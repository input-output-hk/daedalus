## Why now

task-ux-303 shipped the decision-style confirmation modal whose behind-ness line reads
**"Your node is about {count} immutable files behind the latest verified snapshot."**
(`MithrilPartialSyncConfirmation.tsx:27-33`, EN `en-US.json:160`, JA `ja-JP.json:160` literally
"約{count}個のimmutableファイル"). "Immutable files" is an internal/engineering unit, **not** Cardano-user
vocabulary — the exact complaint behind issue #10's copy work (gap #29; user directive 2026-06-24/25;
codified in the orchestrator memory `ux-copy-cardano-vocabulary.md`).

This task replaces that user-facing figure with **Cardano-native language WITHOUT changing the
behind-ness gate**. Per **PRD D12** (supersedes D11's figure *source*), the displayed figure is the
**exact node-tip epoch difference computed in the RENDERER**:

```
behindByEpochs = Math.max(1, networkTip.epoch − localTip.epoch)   // whole-epoch subtraction, no constant
```

— read from the `localTip`/`networkTip` that `DaedalusDiagnostics` already holds and already renders in the
"Last network block" / "Last synchronized block" rows beside the modal. **No** backend `behindByEpochs`
field, **no** per-network `filesPerEpoch` constant, and **no** change to `getPartialSyncBehindness`
(`source/main/mithril/MithrilPartialSyncService.ts:677-706`). The gate (`isSignificantlyBehind` /
`behindByImmutables`) stays
backend-owned and unchanged (D2 reaffirmed). The modal renders **epochs as the primary sentence** then the
**current sync percentage as a SEPARATE trailing reference sentence**, framed to convey the **time/effort
benefit of partial sync vs waiting for normal sync**.

This establishes the single canonical behind-ness copy + node-tip figure later reused by the proactive
prompt (task-ux-302, which sources the epochs figure the **same** way — renderer node-tip difference — from
`SyncingConnectingPage`'s injected `networkStatus`).

### NetworkStatusStore tip/epoch shape (confirmed)

`localTip`/`networkTip` are typed `TipInfo | null | undefined`; `TipInfo.epoch` is a **plain non-nullable
`number`** (`api/network/types.ts:1-5`). They flow `NetworkStatusStore → DaedalusDiagnosticsDialog
(networkStatus.networkTip / .localTip, `:70-71`) → DaedalusDiagnostics props (`:409-410`)`. A missing tip
presents as `null`/`undefined`; Diagnostics already treats `tip && tip.epoch` as the render guard (so
`epoch === 0` falls back to the sand-clock icon). **Match that convention exactly** for the `behindByEpochs`
guard (treat a null/undefined tip, or a non-finite epoch, as "unknown" → render the fallback line). There is
**no** epoch field on the main process / availability read model — confirming D12's rationale that the figure
must be renderer-derived.

## DESIGN DECISIONS (resolved — implementer makes none except pasting the approved copy)

### DD1 — Compute `behindByEpochs` in `DaedalusDiagnostics.render()`, not in the store or backend
Compute it where the tips already live and are already rendered. Place the computation next to
`formattedSyncPercentage` (`:571`):

```ts
const networkEpoch =
  networkTip && Number.isFinite(networkTip.epoch) ? networkTip.epoch : null;
const localEpoch =
  localTip && Number.isFinite(localTip.epoch) ? localTip.epoch : null;
const behindByEpochs =
  networkEpoch !== null && localEpoch !== null
    ? Math.max(1, networkEpoch - localEpoch)
    : undefined;
```

- **Whole-epoch subtraction** (D12 explicit user decision 2026-06-25), NOT
  `round(Δabsolute-slot ÷ epochLength)` — for visual consistency with the epoch numbers in the
  "Last network block" / "Last synchronized block" rows on the same screen.
- **Floor at 1** via `Math.max(1, …)` — node-tip distance ≥ the certified gap, so whenever the gate fires the
  figure is ≥ 1 (never shows 0 epochs while offering; lock #5 below).
- `undefined` when either tip (or its epoch) is unavailable → the modal renders the `behindUnknown` fallback
  (lock: no fabricated number). The renderer performs **no threshold comparison** — `behindByEpochs` is a
  pure display figure (lock #1).

### DD2 — Props: drop `behindByImmutables` display, add `behindByEpochs` + `formattedSyncPercentage`
- `DaedalusDiagnostics` → `MithrilPartialSyncSection`: pass `behindByEpochs={behindByEpochs}` instead of
  `behindByImmutables={mithrilPartialSyncBehindByImmutables}`. `formattedSyncPercentage` is **already
