### Summary
Implemented the live-activity affordances for the shared Mithril progress overlay (partial-sync AND
empty-DB bootstrap, which share `MithrilProgressView` / `MithrilStepIndicator`):

1. **Rotating active top-level step.** `MithrilStepIndicator.TopLevelIcon` active branch now renders the
   existing `spinnerIcon` via `SVGInline` with `classNames(styles.icon, styles.iconSpinner)` instead of
   the static `.activeCircle` dot. No scss edit was needed — `.iconSpinner` (scss L71-77) and the

   already exist, so reduced-motion stays honored.
2. **Populated stopping-node step.** Added `'stopping-node': 'preparing'` as the leading entry in
   `STATUS_TO_STEP`, so `stopping-node` lights up the `preparing` step (active/spinning) instead of three
   greyed placeholders. Inert for bootstrap (bootstrap never has `status === 'stopping-node'`).
3. **Renderer-side ticking elapsed.** Added `@observable startedAt: number | null` to
   `MithrilPartialSyncStore` and the anchor logic in `_updateStatus` (mirrors `MithrilBootstrapStore`):
   reset on idle/terminal→working, stamp on first working frame honoring backend `elapsedSeconds`,
   release on `idle`. Threaded store `startedAt` → overlay `startedAt` prop → `MithrilProgressView`
   `bootstrapStartedAt` (the proven ticking effect). Stopped passing the frozen
   `elapsedSeconds={transferProgress?.elapsedSeconds}` from the overlay and removed the dead
   `elapsedSeconds:` line from the `App.tsx` `transferProgress` literal. The `elapsedSeconds` prop on
   `MithrilProgressView` is kept (not removed) to avoid storybook ripple, per plan.
4. **Freeze elapsed on completed.** Added `'completed'` to `TERMINAL_STATUSES` in `MithrilProgressView`
   so the success/handoff frame freezes elapsed at its last value (both flows; accepted per plan Step 5a —
   `completed` is transient and `starting-node` resumes the live tick from the same anchor).
5. **Long-phase reassurance.** Added `LONG_RUNNING_STATUSES`
   (verifying/unpacking/converting/installing/finalizing); render a one-line reassurance `<p>`
   (`aria-live="polite"`, no `role="status"` to avoid colliding with the start/stop `getByRole('status')`).
6. **Stopping-node reassurance block.** Mirrored the `isStartingNode` block with an `isStoppingNode`
   block (role=status, aria-live=polite, aria-atomic, completion spinner) and two optional override props
   (`stoppingNodeTitle`/`stoppingNodeDetail`) wired from the overlay.
7. **i18n.** Added 5 message ids (EN+JA first-class, no `!!!` in catalogs): bootstrap-scoped
   `progress.longPhaseReassurance`, `progress.nodeStoppingTitle`, `progress.nodeStoppingDetail`; and
   partial-sync-scoped `progress.nodeStoppingTitle`, `progress.nodeStoppingDetail`. Ran `yarn i18n:extract`
   + `yarn i18n:check` to regenerate `translations/messages.json` and `defaultMessages.json`.
